# 추상 클래스란?

- 자바에서는 abstract로 선언된 클래스를 추상 클래스 (abstract class)라고 합니다.
- 중복되거나 공통적인 기능을 추상 클래스로 정의해두면, 이를 상속받는 클래스에서 필요한 부분만 재정의하여 사용할 수 있어 개발 효율성을 높일 수 있습니다.

```
abstract class 클래스 이름 {
    ...
    abstract 반환 타입 메서드 이름();
    ...
}
```

#### 예시

TV 클래스가 있다고 가정해 봅시다. <br />
이를 상속받는 Samsung TV, LG TV 클래스는 전원 ON/OFF, 채널 변경, 소리 조절 등 기본 기능은 동일합니다. <br />
하지만 제조사마다 내부 동작 방식은 다르므로, 추상 클래스로 공통 기능의 틀을 정의하고 세부 동작은 각 제조사 클래스에서 구현하도록 할 수 있습니다. <br />

## 추상 메서드

- 구현(중괄호 `{}`) 없이 선언만 된 메서드입니다.
- 추상 메서드를 포함하는 클래스는 반드시 abstract로 선언해야 합니다.

`abstract 반환 타입 메서드 이름();`

# 추상 클래스의 특징

- `new` 키워드를 사용해서 직접 객체를 생성할 수 없습니다. 추상 클래스를 상속한 하위 클래스에서 객체 생성이 가능합니다.
- 일반 클래스처럼 생성자, 변수, 일반 메서드를 가질 수 있습니다.
- 추상 클래스를 상속받은 하위 클래스는 추상 클래스의 모든 메서드를 구현해야 합니다.
- 다중 상속이 불가능합니다.

## 추상 클래스 구현

```
abstract class Animal {
    // 추상 메서드
    abstract void cry();
}

class Cat extends Animal {
    void cry() {
        System.out.println("야옹야옹");
    }
}

class Dog extends Animal {
    void cry() {
        System.out.println("멍멍");
    }
}

public class Main {
    public static void main(String[] args) {
        // Animal a = new Animal(); // 추상 클래스는 인스턴스 생성 불가

        Cat c = new Cat();
        Dog d = new Dog();

        c.cry(); // "야옹야옹"
        d.cry(); // "멍멍"
    }
}
```

# 인터페이스란?

인터페이스 (interface)는 클래스들을 작성할 때 기본이 되는 틀을 제공하고, 서로 다른 클래스 간의 중간 매개 역할을 하는 일종의 추상 클래스입니다.

```
interface 인터페이스 이름 {
    public static final 타입 상수 이름 = 값;
    ...
    public abstract 메서드 이름(매개변수 목록);
    ...
}
```

# 인터페이스의 특징

- `new` 키워드를 사용해서 직접 객체를 생성할 수 없습니다. 인터페이스를 구현한 하위 클래스에서 객체 생성이 가능합니다.
- 추상 메서드, 상수 (static final) 만으로 구성되어 있습니다.
- 인터페이스를 구현한 하위 클래스는 인터페이스의 모든 추상 메서드를 구현해야 합니다.
- 다중 상속이 가능합니다.

## 인터페이스 구현

```
interface Animal {
    public abstract void cry();
}

class Cat implements Animal {
    public void cry() {
        System.out.println("야옹야옹");
    }
}

class Dog implements Animal {
    public void cry() {
        System.out.println("멍멍");
    }
}

public class Main {
    public static void main(String[] args) {
        Cat c = new Cat();
        Dog d = new Dog();

        c.cry(); // "야옹야옹"
        d.cry(); // "멍멍"
    }
}
```

# 추상 클래스 vs 인터페이스

### 공통점

- 인스턴스화가 불가능하다.
- 가지고 있는 추상 메서드의 구현을 강제한다.

### 차이점

| 구분                 | 추상 클래스                                 | 인터페이스                               |
| -------------------- | ------------------------------------------- | ---------------------------------------- |
| **필드**             | 일반 인스턴스 필드 가능 (static/final 아님) | 모든 변수는 `public static final`        |
| **메서드 접근 제한** | `public`, `protected`, `private` 가능       | 기본적으로 `public`                      |
| **상속/구현**        | 단일 클래스 상속 (`extends`)만 가능         | 여러 인터페이스 구현 (`implements`) 가능 |

# 언제 사용할까?

### 추상 클래스

- 밀접하게 관련된 클래스들 사이에서 코드 공유가 필요할 때
- 하위 클래스들이 공통 메서드나 필드를 많이 가지거나, protected/private 같은 접근 제어자가 필요한 경우
- 객체 상태를 관리하고 싶을 때
  static: 클래스 단위로 공유되는 변수나 메서드를 만들 때 사용합니다.
  final: 한 번 값이 할당되면 바꿀 수 없습니다.

### 인터페이스

- 서로 관련 없는 클래스들이 동일한 기능을 구현해야 할 때 (예: Comparable, Cloneable)
- 다중 상속의 장점을 활용하고 싶을 때
