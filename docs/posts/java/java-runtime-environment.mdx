# JVM, JRE, JDK

Java 개발 환경을 이해하기 위해서는 JVM, JRE, JDK의 개념과 상호 관계를 명확히 알아야 합니다. 이 세 요소는 Java 프로그램의 개발과 실행에 각각 다른 역할을 담당하며, 계층적인 포함 관계를 가지고 있습니다.

## JVM (Java Virtual Machine)

JVM은 Java 바이트코드를 실행하는 가상 머신입니다. Java의 "Write Once, Run Anywhere" 철학을 실현하는 핵심 기술로, 플랫폼에 독립적인 실행 환경을 제공합니다.

### 주요 역할

- **바이트코드 실행** : .class 파일(바이트코드)을 기계어로 변환하여 실행
- **메모리 관리** : 힙, 스택, 메서드 영역 등 메모리 영역을 관리
- **가비지 컬렉션** : 사용하지 않는 객체를 자동으로 메모리에서 해제
- **플랫폼 추상화** : 운영체제별 차이를 추상화하여 동일한 코드가 다양한 환경에서 실행 가능

### 특징

JVM은 스택 기반의 가상 머신으로, 각 운영체제(Windows, Linux, maxOS)마다 고유한 JVM 구현체가 존재합니다. 하지만 모든 JVM은 동일한 바이트코드를 실행할 수 있어 플랫폼 독립성을 보장합니다.

## JRE (Java Runtime Environment)

JRE는 Java 프로그램을 실행하기 위한 최소한의 환경을 제공하는 패키지입니다. 컴파일된 Java 애플리케이션을 실행하는 데 필요한 모든 구성 요소를 포함합니다.

### 구성 요소

- **JVM** : Java 바이트코드를 실행하는 가상 머신
- **Java 클래스 라이브러리** : 표준 API (java.lang, java.util, java.io 등)
- **Java 런타임 라이브러리** : 애플리케이션 실행에 필요한 기본 라이브러리

### 사용 목적

JRE는 이미 컴파일된 Java 프로그램을 실행하기만 하려는 사용자를 위한 환경입니다. 개발 도구는 포함되지 않으므로 Java 프로그램을 작성하거나 컴파일할 수는 없습니다.

## JDK (Java Development Kit)

JDK는 Java 어플리케이션을 개발하고 실행하기 위한 완전한 개발 환경을 제공합니다. Java 프로그램의 작성, 컴파일, 디버깅, 실행까지 모든 과정을 지원합니다.

### 구성 요소

- **JRE** : Java 프로그램 실행 환경 (JVM + 라이브러리)
- **개발 도구** : :
  - `javac` : Java 컴파일러 (.java -> .class)
  - `java` : Java 인터프리터 (실행기)
  - `javadoc` : 문서 생성 도구
  - `jar` : JAR 파일 생성 및 관리 도구
  - `jdb` : Java 디버거
- **추가 유틸리티** : 프로파일링, 모니터링 도구 등

### 사용 목적

JDK는 Java 개발자를 위한 패키지로, Java 애플리케이션의 개발부터 배포까지 전 과정을 지원합니다.

## 세 요소의 관계와 차이점

### 포함 관계

JDK, JRE, JVM은 다음과 같은 계층적 포함 관계를 가집니다 :

JDK (Java Development Kit)
├── JRE (Java Runtime Environment)
│   ├── JVM (Java Virtual Machine)
│   └── Java 클래스 라이브러리
└── 개발 도구 (javac, javadoc, jar 등)

### 역할별 구분

| 구분 | JVM | JRE | JDK |
|------|-----|-----|-----|
| **목적** | 바이트코드 실행 | Java 프로그램 실행 | Java 프로그램 개발 |
| **대상** | 시스템 레벨 | 일반 사용자 | 개발자 |
| **기능** | 가상 머신 | 실행 환경 | 개발 + 실행 환경 |
| **포함 요소** | 가상 머신만 | JVM + 라이브러리 | JRE + 개발 도구 |

### 선택 기준

- **Java 프로그램만 실행**: JRE 설치
- **Java 프로그램 개발**: JDK 설치 (JRE 포함됨)
- **서버 배포**: JRE만으로 충분 (용량 절약)

## 정리

JVM은 Java 바이트코드를 실행하는 가상 머신이고, JRE는 JVM과 필요한 라이브러리를 포함한 실행 환경이며, JDK는 JRE에 개발 도구까지 추가한 완전한 개발 환경입니다. 개발자라면 JDK를 설치하고, 단순히 Java 애플리케이션을 실행만 한다면 JRE만 있으면 됩니다.
