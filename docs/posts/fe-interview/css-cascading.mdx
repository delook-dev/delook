## 답변

CSS 우선순위는 **인라인 > ID > 클래스/속성/가상 클래스 > 요소/가상 요소** 순서로 적용됩니다.
우선 순위가 동일하다면 나중에 작성된 스타일이 적용됩니다.
`!important`가 붙은 스타일은 우선순위를 무시하고 강제 적용됩니다.
작성된 스타일을 읽으면서 충돌사항을 확인하고, 우선순위를 계산하여 최종 스타일을 결정하는 과정을 `cascading`이라고 합니다.

#### 요소/가상 요소 선택자 (우선순위 가장 낮음)

```tsx
<p>요소 선택자 예시</p>

<style>
  p {
    color: blue;
  }
</style>
```

#### 클래스 선택자

`p`보다 `.text`의 우선순위가 높아 텍스트 색상은 `green`으로 적용됨

```tsx
<p class="text">클래스 선택자 예시</p>

<style>
  p {
    color: blue;
  }
  .text {
    color: green;
  }
</style>
```

#### ID 선택자

`.text`보다 `#title`의 우선순위가 높아 텍스트 색상은 `red`로 적용됨

```tsx
<p id="title" class="text">ID 선택자 예시</p>

<style>
  .text {
    color: green;
  }
  #title {
    color: red;
  }
</style>
```

#### 인라인 스타일

인라인 스타일이 모든 선택자보다 우선되므로 텍스트 색상은 `orange`로 적용됨

```tsx
<p id="title" class="text" style="color: orange;">
  인라인 스타일 예시
</p>

<style>
  .text {
    color: green;
  }
  #title {
    color: red;
  }
</style>
```

#### !important

인라인 스타일이 있어도 텍스트 색상은 `green`으로 최종 적용됨

```tsx
<p id="title" class="text" style="color: orange;">
  !important 예시
</p>

<style>
  .text {
    color: green !important;
  }
</style>
```

### cascading

브라우저의 렌더링 엔진이 스타일 충돌을 해결하고, 어떤 CSS 속성을 최종적으로 적용할지를 계산하는 과정입니다.
참고로 부모 요소로부터 상속받은 스타일은 항상 가장 낮은 우선순위를 가지며,
reset css 하지 않은 경우 브라우저에서 적용하는 기본 스타일 역시 우선순위가 낮습니다.

다음의 순서대로 따져본 다음 최종 스타일을 결정하게 됩니다:

1. 중요도(Importance)
2. 명시도(Specificity)
3. 소스 코드 순서(Order)
