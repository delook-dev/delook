## 답변

Flexbox는 1차원 레이아웃 시스템으로 행 또는 열 중 하나의 방향으로만 요소들을 배치합니다. <br />
Grid는 2차원 레이아웃 시스템으로 행과 열을 동시에 다루며, 복잡한 레이아웃 구조를 만드는 데 적합합니다.

### Q. 어떤 기준으로 선택하나요?

하나만 사용하는 게 아니라, Grid로 전체 구조를 짜고 Flexbox로 내부 요소 정렬을 하는 등의 혼합 사용이 많습니다. 각 기술의 장점을 조화롭게 활용하는 것이 중요합니다.

추가적으로 Flexbox와 Grid를 선택하는 기준과 고려해야 하는 요소는 다음과 같습니다:

#### 1. 레이아웃의 복잡성

- 단순히 행 혹은 열 한 방향만 정렬한다면 Flexbox가 적합합니다.
- 행과 열이 모두 필요하고, 레이아웃 구조가 복잡하다면 Grid가 적합합니다.

#### 2. 정렬 및 배치 방식

- 요소들을 `가로·세로 정렬`만 수행하면 되는 내비게이션 바, 버튼 그룹 등에는 Flexbox가 편리합니다.
- 카드 레이아웃, 갤러리, 대시보드처럼 여러 행과 열에 맞춰 요소들을 배치해야 할 때는 Grid가 적합합니다.

#### 3. 반응형 설계 및 유연성

두 레이아웃 방식 모두 반응형에 강점이 있습니다. 하지만, 한 가지만 선택하고 싶다면 선택 기준은 다음과 같습니다:

- 요소의 정렬·간격 조정이 중심이라면 Flexbox가 편리합니다.
- 전체 영역의 구조 변화와 조합이 중심이라면 Grid를 우선 고려하는 것이 좋습니다.


#### 정리

| 핵심 차이점            | Flexbox                                       | Grid                                    |
| ---------------------- | --------------------------------------------- | --------------------------------------- |
| 차원성(Dimensionality) | 1차원 시스템                                  | 2차원 시스템                            |
| 레이아웃 접근 방식     | 콘텐츠 중심적 접근법                          | 레이아웃 중심적 접근법                  |
| 자동 배치 및 정렬      | 자동 크기 조정/간격 분배                      | 명시적 배치/정확한 위치 지정            |
| 주 용도                | 간단한 정렬/버튼 그룹, 수평·수직 단일 행 배치 | 카드 레이아웃, 대형 페이지, 복잡한 구조 |
