웹팩은 모듈 번들러로, 여러 파일들을 하나로 묶어주는 역할을 하며, 바벨은 JavaScript 컴파일러로 최신 문법을 이전 버전의 JavaScript로 변환해주는 역할을 합니다. <br/>
웹팩은 프로젝트의 모든 자원(JS, CSS, 이미지 등)을 관리하고 최적화하며, 바벨은 코드의 호환성을 보장하는 데 중점을 둡니다.

### Q. 웹팩의 주요 기능과 장점은 무엇인가요?

웹팩은 여러 파일을 하나로 묶어 HTTP 요청을 최소화하는 기능을 제공합니다. CommonJS, AMD, ES6 모듈 등 다양한 모듈 시스템을 지원하고, 의존성 트리를 자동으로 분석하고 관리합니다.

또한 로더(Loader)를 통해 JavaScript, CSS, 이미지 등 다양한 파일 타입별로 적절한 변환을 수행할 수 있습니다. 플러그인(Plugin)을 사용하면 번들 최적화, 환경 변수 주입, HTML 생성 등 번들링 과정에서 추가 작업을 수행할 수 있습니다.

특히 코드 스플리팅 기능은 필요한 코드만 로드하여 초기 로딩 시간을 단축하고, 동적 임포트를 통한 지연 로딩을 지원하며, 공통 모듈 분리를 통한 캐시 효율성을 향상시킵니다.

### Q. 바벨의 주요 기능에 대해 설명해주세요.

바벨은 JavaScript 컴파일러로, 주로 트랜스파일링을 담당합니다. ES6+ 문법을 ES5로 변환하고, JSX를 일반 JavaScript로 변환하며, TypeScript를 JavaScript로 변환하는 작업을 수행합니다.

폴리필 기능도 중요한데, 이는 새로운 JavaScript 기능을 이전 버전에서도 사용할 수 있게 해줍니다. core-js를 통한 기능 구현과 regenerator-runtime을 통한 async/await 지원이 대표적인 예시입니다.

또한 플러그인과 프리셋을 통해 개별 문법 변환을 수행하고, 여러 플러그인을 모아놓은 프리셋을 사용할 수 있으며, 필요한 경우 커스텀 변환 규칙을 정의할 수 있습니다.
