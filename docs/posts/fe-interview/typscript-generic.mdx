## 답변
`제네릭(Generic)`이란 타입을 함수의 파라미터처럼 사용하는 것을 의미합니다. <br />
즉, 타입을 직접적으로 고정된 값으로 명시하지 않고 타입을 변수로 지정해서 언제든지 변할 수 있는 타입을 보장합니다. <br />
제네릭은 함수가 선언되는 시점이 아닌 호출 시점에 타입을 결정하게 됩니다. <br />
따라서, 같은 함수라도 여러 타입의 인자를 받는 경우 재사용성을 높일 수 있어 유용합니다.

#### 제네릭 기본 문법
```ts
function printSentence<T>(sentence: T): T {
    return sentence;
}
```

타입 파라미터로 쓰인 대문자 `T`는 Type을 의미하며 임의의 다른 문자로 대체 가능합니다. <br />
(자주 쓰이는 타입 파라미터 컨벤션 예시: E(Element), K(Key), V(Value) 등)


제네릭 함수 호출 시 다음과 같이 사용할 수 있습니다.
```ts
printSentence('hello, delook') // 타입 추론을 이용하는 경우

// 할당할 타입을 직접 명시하는 경우
printSentence<string>('hello, delook')
printSentence<number>(100) 
```

#### 제네릭이 필요한 경우
제네릭이 없다면 같은 함수라고 여러 가지 타입의 인자를 받거나 반환하는 경우 `any` 타입 또는 `유니온(Union)` 타입을 할당할 수 있습니다. <br />
하지만, `any` 타입을 할당하는 경우 타입 체크를 하지 않게 됩니다. <br />
`유니온` 타입을 할당하는 경우 다중 타입이 가능하지만 특정 타입에서만 동작 가능한 메서드의 사용이 불가능해집니다.

```ts
function printSentence(sentence: string | number): string | number {
  return sentence;
}

const apple = printSentence("apple")
apple.toUpperCase() // X : Property 'toUpperCase' does not exist on type 'string | number'.
```

이처럼 `유니온 타입`으로 특정 타입의 연산 메소드를 이용하려는 경우 `타입 좁히기(type narrowing)`가 필요합니다.

```ts
if (typeof(apple) === 'string') {
  apple.toUpperCase()
}
```

이런 경우 제네릭 타입을 사용하면 함수를 호출할 때 타입이 그대로 보존되어 있어서 타입별 고유 기능을 안전하게 사용할 수 있고, <br />
코드의 길이를 줄여 가독성을 높여줍니다.
