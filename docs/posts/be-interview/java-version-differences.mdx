## 답변

대표적인 LTS 버전은 8, 11, 17, 21이며 각 버전은 언어 기능과 JVM 성능 측면에서 중요한 차이가 있습니다.

- Java 8에서는 람다 표현식, Stream API, Optional 클래스가 도입되어 본격적인 함수형 프로그래밍이 가능해졌습니다.
- Java 11에서는 Http Client 표준화, var 키워드, 문자열 관련 메서드가 추가되었으며, 저지연 가비지 컬렉터(ZGC)가 도입되었습니다.
- Java 17에서는 불변 데이터 클래스 Record, 계층형 클래스 Sealed Class, Switch Expressions 기능이 확장되었습니다.
- Java 21에서는 Virtual Threads, Switch에서의 Pattern Matching, Record Patterns를 이용한 구조 분해가 가능해졌으며, GC 성능도 개선되었습니다.

---

### Java 8

- 람다 표현식(Lambda Expressions) 도입
- Stream API 지원
- java.time 패키지(새로운 Date/Time API)
- Optional 클래스 도입
- 병렬 처리와 함수형 프로그래밍 지원 강화

#### 예시

```java
List<String> names = Arrays.asList("Apple", "Banana", "Cherry", null, "Durian");

// Stream + Lambda + Optional
names.stream()
    .map(name -> Optional.ofNullable(name)) // null 안전하게 Optional로 감싸기
    .filter(Optional::isPresent)            // 값이 있는 것만 통과
    .map(Optional::get)                     // Optional에서 실제 값 꺼내기
    .map(String::toUpperCase)               // Lambda: 대문자로 변환
    .forEach(System.out::println);          // 결과 출력
```

실행 결과

```
APPLE
BANANA
CHERRY
DURIAN
```

### Java 11

- `var` 키워드를 통한 지역 변수 타입 추론 (Java 10에서 도입, LTS로 정착)
- HTTP Client API (java.net.http) 표준화
- String 메서드 추가 (`isBlank()`, `lines()`, `repeat()` 등)
- ZGC(Z Garbage Collector) 도입
- Nashorn JavaScript Engine 제거

#### 예시

```java
var message = "Hello Java 11";
System.out.println(message.isBlank()); // false
```

### Java 17

- Sealed Classes (클래스 계층 제어)
- Pattern Matching for instanceof
- Record (불변 데이터 클래스)
- Switch Expressions `->` 추가 (Java 12에서 미리보기 → 정식 반영)
- JDK 내부 API 강력 캡슐화

#### 예시

```Java
// sealed , record 예시
sealed interface Shape permits Circle, Rectangle {}

record Circle(double radius) implements Shape {}
record Rectangle(double width, double height) implements Shape {}

Shape shape = new Circle(5.0);

// Pattern Matching for instanceof
if (shape instanceof Circle c) { // 체크와 동시에 타입 캐스팅 → c 변수로 바로 접근 가능
    System.out.println(c.radius());
}

// Switch Expression -> 예시
Day day = Day.WEDNESDAY;
String dayType = switch (day) {
    case MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY -> "Weekday";
    case SATURDAY, SUNDAY -> "Weekend";
};
System.out.println(day + " is a " + dayType); // Output: WEDNESDAY is a Weekday

```

실행 결과

```
5.0
WEDNESDAY is a Weekday
```

### Java 21

- Virtual Threads (Project Loom, 경량 스레드)
- Pattern Matching for switch
- Record Patterns
- Sequenced Collections API
- String Template (미리보기 기능)
- 성능 최적화 및 최신 GC 개선

#### 예시

```Java
sealed interface Shape permits Circle, Rectangle {}

record Circle(double radius) implements Shape {}
record Rectangle(double width, double height) implements Shape {}

Shape shape = new Rectangle(4.0, 6.0);

// Switch + Record Pattern
String desc = switch (shape) { // switch 문에서 pattern matching
    case Circle(double r) -> "Circle with radius " + r;
    case Rectangle(double w, double h) -> "Rectangle " + w + "x" + h; // Record Pattern으로 구조분해
    default -> "Unknown shape";
};

System.out.println(desc);
```

실행 결과

```
Rectangle 4.0x6.0
```
