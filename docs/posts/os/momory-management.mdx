# Paging과 Segmentation

## 페이징(Paging)이란?

페이징은 하나의 프로세스가 사용하는 메모리 공간이 반드시 연속적이어야 한다는 제약을 없앤 가상 메모리 관리 기법입니다.  
논리 메모리는 페이지(Page)라 불리는 고정 크기의 블록으로 분리되고, 각 페이지는 물리 메모리의 프레임(Frame)에 할당됩니다. 페이지와 프레임은 동일한 크기로 정의됩니다.

논리 메모리는 물리 메모리에 저장될 때 연속적으로 배치되지 않아도 되며, 빈 프레임에 적절히 배치될 수 있습니다. 이때 각 페이지가 물리 메모리 어디에 저장되었는지를 관리하기 위해 페이지 테이블(Page Table)이 사용됩니다.  
페이지 테이블은 가상 주소 공간을 실제 물리적 메모리 공간에 매핑(mapping)하는 자료구조로, 페이지 번호와 프레임 번호를 연결해줍니다.


## 페이징의 장단점

### 장점
1. **외부 단편화(External Fragmentation) 해결**  
   - 외부 단편화란, 가변 크기 블록을 사용하는 세그멘테이션 방식에서 빈 공간이 여러 곳으로 나뉘어 생기는 현상을 의미합니다.  
   - 전체 메모리에 여유 공간은 있지만 연속된 공간이 부족하여 큰 프로세스를 배치하지 못하는 상황이 발생할 수 있습니다.  
   - 페이징은 고정 크기 블록을 사용하기 때문에 이러한 문제를 방지할 수 있습니다.

2. **논리 주소 공간 확대**  
   - 논리 주소와 물리 주소를 분리하여, 물리 메모리 크기보다 더 큰 주소 공간을 사용할 수 있습니다.

3. **메모리 관리 용이**  
   - 가상 메모리를 효과적으로 활용할 수 있으며, 프로세스의 연속적인 배치를 요구하지 않습니다.

### 단점
1. **내부 단편화(Internal Fragmentation) 발생**  
   - 고정 크기의 페이지 단위로 메모리를 할당하기 때문에, 실제 필요한 메모리보다 큰 공간이 배정될 수 있습니다.  
   - 이로 인해 사용되지 않는 메모리 공간이 생기며 낭비가 발생합니다.

2. **주소 변환 오버헤드**  
   - 페이지 테이블을 통해 주소를 변환하는 과정에서 추가적인 오버헤드가 발생합니다.  
   - 이를 완화하기 위해 TLB(Translation Lookaside Buffer)와 같은 캐시 기법이 사용됩니다.


## 세그멘테이션(Segmentation)이란?

세그멘테이션은 페이징과 달리 메모리를 동일한 크기의 블록이 아닌 **가변 크기의 논리적 단위**로 분할하여 관리하는 가상 메모리 기법입니다.  
프로그램은 코드, 전역 변수, 힙, 스택, 라이브러리 등과 같은 **논리적 단위**로 나뉘며, 각 단위는 개별적인 세그먼트(Segment)로 관리됩니다.  
각 세그먼트는 크기가 서로 다를 수 있으며, 메모리 내에서 서로 다른 위치에 배치될 수 있습니다.


## 세그멘테이션의 장단점

### 장점
1. **내부 단편화 해결**  
   - 필요한 크기만큼 메모리를 동적으로 할당하기 때문에, 고정 크기 단위에서 발생하는 내부 단편화를 줄일 수 있습니다.

2. **논리적 구조 반영**  
   - 프로그램의 논리적 구조를 메모리에 그대로 반영할 수 있어 관리가 유연합니다.  
   - 동일한 코드 세그먼트를 여러 프로세스가 공유할 수 있어 자원의 효율적인 활용이 가능합니다.

### 단점
1. **외부 단편화 발생**  
   - 세그먼트의 크기가 서로 다르기 때문에, 메모리의 빈 공간이 여러 조각으로 나뉘어 외부 단편화가 생길 수 있습니다.

2. **스왑과 관리 비용 증가**  
   - 세그먼트를 스왑할 때 다양한 크기를 고려하여 스왑 영역을 확보해야 하므로 부담이 커집니다.  
   - 세그먼트 테이블을 관리해야 하므로 추가적인 비용이 발생합니다.  
   - 또한 동적 메모리 할당 시 세그먼트 크기를 조정해야 하는 문제가 생길 수 있습니다.

## Paging vs Segmentation 비교

| 구분 | 페이징(Paging) | 세그멘테이션(Segmentation) |
|------|----------------|-----------------------------|
| 단위 | 고정 크기 페이지(Page) | 가변 크기 세그먼트(Segment) |
| 주소 변환 방식 | 페이지 번호 + 오프셋 | 세그먼트 번호 + 오프셋 |
| 단편화 | 내부 단편화 발생 | 외부 단편화 발생 |
| 관리 구조 | 페이지 테이블 필요 | 세그먼트 테이블 필요 |
| 논리적 구조 반영 | 어려움 (균일 분할) | 용이 (코드, 데이터, 스택 등 반영) |
| 공유 및 보호 | 페이지 단위 | 세그먼트 단위 |
| 장점 | 외부 단편화 해소, 관리 단순 | 내부 단편화 해소, 논리적 구조 반영 |
| 단점 | 내부 단편화, 주소 변환 오버헤드 | 외부 단편화, 관리 복잡성 증가 |
