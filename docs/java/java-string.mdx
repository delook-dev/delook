# JAVA의 String, StringBuilder, StringBuffer

JAVA에서는 문자열 변수를 다루기 위한 여러 클래스가 존재합니다.
이 문서에서는 String, StringBuilder, StringBuffer 클래스에 대해 학습합니다.
각 클래스의 개념과 차이점, 특징을 학습합니다.

## String
- `java.lang.String` 클래스는 불변(immutable) 객체로, 한 번 생성하면 내부 값을 변경할 수 없습니다.
- 문자열 리터럴을 다루는 데 가장 널리 쓰이는 타입입니다.

### String의 주요 특징
| 항목          | 설명                                            |
| ----------- | --------------------------------------------- |
| 불변성         | 생성된 후 내용 변경 불가                                |
| String Pool | 같은 문자열 리터럴은 JVM 내에서 공유                        |
| 사용 편의성      | `+`, `.concat()`, `.substring()` 등 편리한 API 제공 |
| 메모리 상 노출    | GC 되기 전까지 민감 정보가 메모리에 남음                      |
| 동기화         | 불필요 (immutable이므로 thread-safe)                |

### String의 장점
- 사용이 직관적이며 가독성이 좋습니다.
- 불변성 덕분에 `HashMap`, `Set` 등의 자료구조의 Key로 사용하기 적합합니다.

### String의 단점
- 메모리에 남은 정보를 삭제할 수 없어 민감 정보 저장에 부적절합니다.
- 빈번한 문자열 수정 시, 새로운 객체가 계속 생성되므로 성능 저하가 발생합니다.

## StringBuilder
- `java.lang.StringBuilder`는 가변적인 문자열을 다루기 위한 클래스입니다.
- 내부 `char[]` 버퍼를 사용하여 문자열을 동적으로 변경 가능합니다.

### StringBuilder 특징
| 항목      | 설명                                            |
| ------- | --------------------------------------------- |
| 가변성     | `.append()`, `.delete()`, `.setCharAt()` 등 제공 |
| 싱글스레드용  | **동기화 지원 X** → 빠름                             |
| 효율적 메모리 | 문자열을 많이 조작할 때 효율적                             |

### StringBuilder 장점
- 문자열 결합 시 성능이 우수합니다. (새로운 객체를 생성하지 않고, 기존 객체 내부에서 작업합니다.)
- 내부 값을 삭제하거나 초기화할 수 있습니다. (`setLength(0)` 등의 메서드를 활용할 수 있습니다.)

### StringBuilder 단점
- 멀티스레드 환경에서는 동기화 이슈가 존재합니다.
  - 내부 메서드(append, insert, delete 등)가 동기화 되어 있지 않아, 여러 스레드에서 동시에 접근하면 데이터가 충돌하거나 오작동이 발생할 수 있습니다.
- 내부적으로 `char[]` 배열을 사용하지만, `toString()`을 호출하면 새로운 String 객체가 생성되며 해당 문자열은 힙 메모리에 남아있게 됩니다.
  - 즉, `toString()` 결과로 생긴 String 객체는 GC 전까지 메모리에 평문으로 존재하게 된다는 단점이 있습니다.
 
## StringBuffer
- StringBuilder와 거의 동일하지만, 동기화(Synchronized)를 지원합니다.
- 멀티스레드 환경에서 안전하게 사용 가능합니다.

### StringBuffer 특징
| 항목     | 설명                             |
| ------ | ------------------------------ |
| 가변성    | 내부 버퍼를 수정할 수 있음                      |
| 동기화 지원 | 멀티 스레드에서 thread-safe           |
| 성능     | `StringBuilder`보다 느림 (락 사용 때문) |

### StringBuffer 장점
- 멀티스레드 환경에서 문자열을 조작할 때 안전합니다.
  - 여러 스레드가 동시에 접근해도 동기적으로 처리됩니다.
- CPU 리소스를 조금 더 쓰더라도 안정성을 보장합니다.
- 보안 처리와 동기화가 동시에 필요한 경우에 유용합니다.

### StringBuffer 단점
- 락(lock)때문에 속도가 느릴 수 있습니다.
- 단일 스레드 환경에서는 불필요한 성능 손해가 있습니다.
  - 대부분의 경우에서 StringBuilder로 대체 가능합니다.
 
## 정리
- 평범한 문자열은 String 또는 StringBuilder를 사용합니다.
- 민감 정보(비밀번호 등)은 절대 String에 담지 말고, char[] 또는 상황에 따라 StringBuilder를 사용합니다.
- 동기화 필요 여부에 따라 StringBuffer를 고려합니다.
